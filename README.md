# Навигация робота в лабиринте с помощью машинного обучения

## Цель проекта

Цель проектной работы — реализовать интеллектуального агента, который учится находить выход из лабиринта с препятствиями с помощью алгоритма DQN (Deep Q-Network).

Проект является учебной симуляцией лабиринта, похожей по идее на задачи Gym-MiniWorld, но намного проще для установки и воспроизведения на обычном компьютере. Здесь не используется 3D-среда: лабиринт представлен двумерной сеткой 8x8.

## Что такое обучение с подкреплением

Обучение с подкреплением — это подход в машинном обучении, при котором агент учится действовать в среде методом проб и ошибок.

В этом проекте:

- агент — робот в лабиринте;
- среда — двумерная карта со стенами;
- действие — движение вверх, вниз, влево или вправо;
- награда — численная оценка результата действия;
- цель — научиться получать максимальную суммарную награду, то есть быстро доходить до выхода.

Агент не получает заранее готовый маршрут. Он сам пробует разные действия, запоминает опыт и постепенно улучшает стратегию.

## Что такое DQN

DQN, или Deep Q-Network, — это алгоритм обучения с подкреплением, в котором нейронная сеть приближает Q-функцию.

Q-функция показывает, насколько полезно выполнить конкретное действие в конкретном состоянии. Если Q-значение действия большое, агент считает это действие перспективным.

В проекте DQN использует несколько важных идей:

1. Replay buffer — память прошлых переходов. Агент хранит опыт и обучается на случайных примерах из памяти.
2. Epsilon-greedy — стратегия выбора действий. Иногда агент выбирает случайный ход для исследования лабиринта, а иногда выбирает лучший ход по мнению нейросети.
3. Target network — отдельная целевая сеть. Она помогает сделать обучение стабильнее.
4. PyTorch — библиотека для создания и обучения нейронной сети.

## Описание среды лабиринта

Среда реализована в файле `maze_env.py` в стиле Gymnasium API.

Основные методы:

- `reset()` — начать новый эпизод;
- `step(action)` — выполнить действие агента;
- `render()` — вывести лабиринт в консоль;
- `observation_space` — описание формы состояния;
- `action_space` — описание доступных действий.

Лабиринт имеет размер 8x8. Обозначения при выводе:

```text
A — агент
G — цель
# — стена
. — свободная клетка
```

Доступные действия:

```text
0 — вверх
1 — вниз
2 — влево
3 — вправо
```

Наблюдение для нейросети состоит из трёх каналов:

1. карта стен;
2. позиция агента;
3. позиция цели.

То есть нейросеть получает не только координаты робота, но и структуру лабиринта.

## Система наград

В проекте используется простая система наград:

- `+10.0` — агент дошёл до цели;
- `-1.0` — агент ударился о стену или вышел за границу;
- базовый штраф `-0.05` — за каждый шаг, чтобы агент не ходил слишком долго;
- дополнительная учебная подсказка: если агент приближается к цели, награда немного увеличивается, а если отдаляется — уменьшается.

Такая система наград называется reward shaping. Она помогает DQN быстрее научиться, потому что агент получает не только финальную награду за выход, но и промежуточный сигнал о том, становится ли он ближе к цели.

Такой подход заставляет агента:

- избегать стен;
- не ходить слишком долго;
- постепенно двигаться в сторону цели;
- искать короткий путь к выходу.

Если агент не достигает цели за лимит шагов, эпизод завершается.

## Установка зависимостей

Рекомендуется использовать виртуальное окружение Python 3.10+.

```bash
python -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt
```

Если на компьютере установлен `uv`, можно установить зависимости так:

```bash
uv venv --python 3.12 .venv
source .venv/bin/activate
uv pip install -r requirements.txt
```

## Обучение модели

Для запуска обучения:

```bash
python train.py --episodes 500
```

Можно указать другое число эпизодов, например 300 или 1000:

```bash
python train.py --episodes 300
python train.py --episodes 1000
```

Во время обучения программа выводит прогресс:

```text
Эпизод   10/500 | награда ... | шаги ... | epsilon ... | loss ...
```

После обучения сохраняются:

```text
models/dqn_maze.pth
results/rewards.png
results/episode_lengths.png
```

В коде зафиксирован `random_seed`, поэтому запуск легче воспроизвести.

## Проверка модели

После обучения запустите инференс:

```bash
python inference.py
```

Чтобы убрать паузу между шагами:

```bash
python inference.py --delay 0
```

Программа загрузит `models/dqn_maze.pth`, покажет пошаговое прохождение лабиринта в консоли и в конце выведет:

- дошёл агент до цели или нет;
- количество шагов;
- итоговую награду.

## Какие результаты получены

При обучении на 300–1000 эпизодах агент обычно начинает находить путь к цели значительно лучше, чем случайная стратегия. На графике `results/rewards.png` можно увидеть рост суммарной награды, а на графике `results/episode_lengths.png` — изменение длины эпизодов.

Результаты могут немного отличаться на разных компьютерах, потому что PyTorch и аппаратное ускорение могут давать небольшую недетерминированность. Для учебной проектной работы это нормально.

## Комментарии к основным файлам

- `maze_env.py` — реализация лабиринта, правил движения и наград.
- `dqn_agent.py` — нейросеть, replay buffer, epsilon-greedy, target network и обновление весов.
- `train.py` — полный цикл обучения и сохранение графиков.
- `inference.py` — загрузка обученной модели и демонстрация прохождения.
- `utils.py` — фиксация seed, создание папок и построение графиков.

В коде добавлены комментарии простым русским языком, чтобы студент мог объяснить работу преподавателю.

## Вывод по проекту

В проекте реализован DQN-агент, который обучается навигации в лабиринте с препятствиями. Среда оформлена по принципам Gymnasium, агент написан на PyTorch, есть обучение, сохранение весов, инференс и графики результатов.

Проект показывает, как методы обучения с подкреплением можно применять для задачи навигации робота. Упрощённая 2D-среда делает проект удобным для запуска и сдачи, а логика похожа на более сложные симуляции робототехники.
